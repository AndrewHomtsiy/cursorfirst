Base



  • git config --global user.name blabla - встановлює ім'я юзера, яке буде відображатися у репозиторії.

    • git config --global user.email blabal@bla - показує мейл аккаунт юзера, яким було створено зміни в репозиторії
    git init - створення нового репозиторія git.
    git status - відображає статус файлів.
    git add - підготовлює файли до встановлення
    git commit - заливає дані в репозиторій
    git log - чекнути логи в репозиторію
    git checkout- юзається для переходу з одної гілки на іншу
    git branch - відображає список гілок проекту
    git push - заливає файли в мастер бренч
    git pull - залити останні зміни в репозиторій з певного бренчу (певного юзера)
    git merge - обєднує бренчі
    git clone - копіює дані
    .gitignore - команда яка ігнорує відображення змін в певних файлах

    Advanced

    git revert - скасувати коміт.
    git reset - повернення до попереднього коміту.
    git rebase - команда дозволяє взяти всі зміни, які потрапили в комміті на одній з гілок, і повторити їх на інший.
    git cherry-pick - використовується для перенесення комітів з одного місця репозиторію в інше
    git stash - бере ваші зміни, підготовлені та непідготовлені до комітів, зберігає їх для подальшого використання
